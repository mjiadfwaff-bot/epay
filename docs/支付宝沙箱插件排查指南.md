# 支付宝沙箱插件 - 错误排查指南

## 已修复问题

### 问题：500错误，没有日志

**原因分析**：
1. 沙箱插件直接调用 `alipay_plugin` 类的静态方法
2. `alipay_plugin` 类可能未被系统自动加载
3. PHP找不到类导致500错误
4. 没有异常捕获和日志记录

**解决方案**：
已添加以下功能：

1. **自动加载机制**
```php
static private function loadAlipayPlugin(){
    if(!class_exists('alipay_plugin')){
        $alipay_plugin_file = PLUGIN_ROOT.'alipay/alipay_plugin.php';
        if(file_exists($alipay_plugin_file)){
            require_once($alipay_plugin_file);
            self::writeLog("[加载] 成功加载 alipay_plugin.php");
        }else{
            self::writeLog("[错误] alipay_plugin.php 文件不存在");
            throw new Exception('支付宝插件文件不存在，请确保已安装 alipay 插件');
        }
    }
}
```

2. **异常捕获和日志**
```php
static public function submit(){
    try{
        self::loadAlipayPlugin();
        self::writeLog("[调用] submit()");
        return alipay_plugin::submit();
    }catch(Exception $e){
        self::writeLog("[异常] submit() - ".$e->getMessage());
        return ['type'=>'error','msg'=>'支付宝沙箱插件调用失败：'.$e->getMessage()];
    }
}
```

3. **详细日志记录**
   - 每个方法调用都记录日志
   - 捕获所有异常并记录到日志
   - 日志位置：`plugins/alipaysandbox/inc/log/YYYY-MM-DD.log`

## 常见错误

### 错误1：支付宝插件文件不存在

**日志**：
```
[错误] alipay_plugin.php 文件不存在: /path/to/plugins/alipay/alipay_plugin.php
```

**原因**：系统中未安装官方 `alipay` 插件

**解决**：
1. 检查 `plugins/alipay/` 目录是否存在
2. 如果不存在，从源码安装官方支付宝插件
3. 确保 `plugins/alipay/alipay_plugin.php` 文件存在

---

### 错误2：class not found

**日志**：
```
[异常] submit() - Class 'alipay_plugin' not found
```

**原因**：`loadAlipayPlugin()` 方法未被调用或加载失败

**解决**：
1. 检查 `PLUGIN_ROOT` 常量是否正确定义
2. 确认 `plugins/alipay/alipay_plugin.php` 文件路径正确
3. 查看日志中是否有 `[加载]` 记录

---

### 错误3：网关地址未使用沙箱

**症状**：使用沙箱APPID但提示"应用未签约"

**原因**：config.php 中的 gateway_url 配置错误

**检查**：
```bash
grep "gateway_url" plugins/alipaysandbox/inc/config.php
```

**应该看到**：
```php
'gateway_url' => "https://openapi-sandbox.dl.alipaydev.com/gateway.do",
```

**如果是生产网关**：
```php
'gateway_url' => "https://openapi.alipay.com/gateway.do",  // 错误！
```

**解决**：
修改 `plugins/alipaysandbox/inc/config.php` 第19行为沙箱网关

---

### 错误4：无日志输出

**症状**：`plugins/alipaysandbox/inc/log/` 目录为空

**原因**：
1. 目录权限不足
2. 日志写入失败

**解决**：
```bash
# 检查目录权限
ls -la plugins/alipaysandbox/inc/

# 如果log目录不存在或权限不对，手动创建
mkdir -p plugins/alipaysandbox/inc/log
chmod 755 plugins/alipaysandbox/inc/log

# 测试写入
echo "test" > plugins/alipaysandbox/inc/log/test.log
```

---

### 错误5：签名验证失败

**日志**：
```
[异常] notify() - 验证签名失败
```

**原因**：
1. 支付宝公钥配置错误
2. 应用私钥配置错误
3. 使用了生产环境的密钥

**解决**：
1. 确认使用的是沙箱支付宝公钥
2. 检查密钥格式（去掉头尾和换行）
3. 在沙箱页面重新生成密钥对

## 调试技巧

### 1. 启用PHP错误显示

编辑 `php.ini` 或在入口文件添加：
```php
error_reporting(E_ALL);
ini_set('display_errors', 1);
```

### 2. 查看PHP错误日志

```bash
tail -f /var/log/php_errors.log
# 或
tail -f /var/log/apache2/error.log
# 或
tail -f /var/log/nginx/error.log
```

### 3. 手动触发插件方法

创建测试脚本 `test_sandbox.php`：
```php
<?php
define('PLUGIN_ROOT', __DIR__.'/plugins/');
require 'plugins/alipaysandbox/alipaysandbox_plugin.php';

try{
    $plugin = new alipaysandbox_plugin();
    var_dump($plugin::$info);
    echo "插件加载成功\n";
}catch(Exception $e){
    echo "错误: ".$e->getMessage()."\n";
}
```

运行：
```bash
php test_sandbox.php
```

### 4. 对比官方插件

如果沙箱插件有问题，对比官方插件是否正常：
```bash
# 测试官方插件
grep -r "class alipay_plugin" plugins/alipay/

# 对比配置
diff plugins/alipay/inc/config.php plugins/alipaysandbox/inc/config.php
```

## 验证插件安装

### 检查清单

- [ ] `plugins/alipay/` 目录存在
- [ ] `plugins/alipay/alipay_plugin.php` 文件存在
- [ ] `plugins/alipaysandbox/` 目录存在
- [ ] `plugins/alipaysandbox/alipaysandbox_plugin.php` 文件存在
- [ ] `plugins/alipaysandbox/inc/config.php` gateway_url 为沙箱地址
- [ ] `plugins/alipaysandbox/inc/log/` 目录可写
- [ ] 通道配置正确（沙箱APPID、密钥）

### 快速验证脚本

```bash
#!/bin/bash
echo "=== 支付宝沙箱插件验证 ==="

# 检查目录
if [ -d "plugins/alipay" ]; then
    echo "✓ alipay 插件目录存在"
else
    echo "✗ alipay 插件目录不存在"
fi

if [ -f "plugins/alipay/alipay_plugin.php" ]; then
    echo "✓ alipay_plugin.php 文件存在"
else
    echo "✗ alipay_plugin.php 文件不存在"
fi

if [ -d "plugins/alipaysandbox" ]; then
    echo "✓ alipaysandbox 插件目录存在"
else
    echo "✗ alipaysandbox 插件目录不存在"
fi

if [ -f "plugins/alipaysandbox/alipaysandbox_plugin.php" ]; then
    echo "✓ alipaysandbox_plugin.php 文件存在"
else
    echo "✗ alipaysandbox_plugin.php 文件不存在"
fi

# 检查网关配置
if grep -q "openapi-sandbox.dl.alipaydev.com" plugins/alipaysandbox/inc/config.php; then
    echo "✓ 沙箱网关配置正确"
else
    echo "✗ 沙箱网关配置错误"
fi

# 检查日志目录
if [ -d "plugins/alipaysandbox/inc/log" ]; then
    echo "✓ 日志目录存在"
    if [ -w "plugins/alipaysandbox/inc/log" ]; then
        echo "✓ 日志目录可写"
    else
        echo "✗ 日志目录不可写"
    fi
else
    echo "✗ 日志目录不存在"
fi

echo "=== 验证完成 ==="
```

## 紧急修复

如果遇到严重问题无法解决：

### 方案1：重新安装

```bash
# 备份现有配置
cp plugins/alipaysandbox/inc/config.php /tmp/config_backup.php

# 删除插件
rm -rf plugins/alipaysandbox

# 重新创建（按照README.md操作）
mkdir -p plugins/alipaysandbox/inc/log
mkdir -p plugins/alipaysandbox/cert

# 恢复配置
cp /tmp/config_backup.php plugins/alipaysandbox/inc/config.php
```

### 方案2：使用官方插件

临时方案：直接使用官方 `alipay` 插件，在 `config.php` 中修改网关为沙箱地址
