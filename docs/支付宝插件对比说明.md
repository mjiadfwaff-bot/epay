# 支付宝插件对比说明

## 两个插件的关系

| 插件 | 名称 | 用途 | 网关地址 |
|------|------|------|----------|
| `alipay` | 支付宝官方支付 | **生产环境** | https://openapi.alipay.com/gateway.do |
| `alipaysandbox` | 支付宝沙箱支付 | **测试环境** | https://openapi-sandbox.dl.alipaydev.com/gateway.do |

---

## 核心区别

### 1. 网关地址

```php
// alipay (生产版)
'gateway_url' => "https://openapi.alipay.com/gateway.do"

// alipaysandbox (沙箱版)
'gateway_url' => "https://openapi-sandbox.dl.alipaydev.com/gateway.do"
```

### 2. 配置说明

```php
// alipay (生产版)
'showname' => '支付宝官方支付'
'link' => 'https://b.alipay.com/signing/productSetV2.htm'
'note' => '<p>选择可用的接口...'

// alipaysandbox (沙箱版)
'showname' => '支付宝沙箱支付'
'link' => 'https://open.alipay.com/develop/sandbox/app'
'note' => '<p><font color="red">【沙箱环境专用】用于开发测试...'
```

### 3. 代码逻辑

**完全相同** - `alipaysandbox` 的所有方法都直接调用 `alipay_plugin` 的对应方法：

```php
// alipaysandbox 的所有方法实现
static public function submit(){
    return alipay_plugin::submit();
}

static public function notify(){
    return alipay_plugin::notify();
}

static public function refund($order){
    return alipay_plugin::refund($order);
}
// ... 其他方法类似
```

## 使用场景

### 开发测试阶段
✅ 使用 `alipaysandbox`
- 申请沙箱APPID（免费，即时获得）
- 使用沙箱买家账号测试（虚拟余额）
- 不会产生真实交易
- 网关：`https://openapi-sandbox.dl.alipaydev.com/gateway.do`

### 生产上线阶段
✅ 使用 `alipay`
- 申请正式商户APPID（需企业资质）
- 真实用户付款
- 产生真实交易
- 网关：`https://openapi.alipay.com/gateway.do`

## 配置对比

| 配置项 | alipay (生产) | alipaysandbox (沙箱) |
|--------|---------------|---------------------|
| APPID获取 | 企业认证后申请 | 沙箱页面直接获取 |
| 密钥要求 | RSA2生产密钥 | RSA2沙箱密钥 |
| 买家账号 | 真实用户 | 沙箱测试账号 |
| 交易金额 | 真实扣款 | 虚拟金额 |
| 审核要求 | 需要企业认证 | 无需审核 |
| 证书目录 | /plugins/alipay/cert/ | /plugins/alipaysandbox/cert/ |
| 日志目录 | /plugins/alipay/inc/log/ | /plugins/alipaysandbox/inc/log/ |

## 迁移指南

### 从沙箱迁移到生产

**步骤1：申请生产APPID**
1. 登录 https://open.alipay.com
2. 创建应用（需企业认证）
3. 签约所需产品（如"手机网站支付"）

**步骤2：生成生产密钥**
1. 使用密钥工具生成新的RSA2密钥对
2. 上传应用公钥到生产应用
3. 下载生产支付宝公钥

**步骤3：配置新通道**
1. 后台添加支付通道
2. 选择 `alipay`（支付宝官方支付）
3. 填入生产APPID和密钥
4. 启用生产通道，禁用沙箱通道

**步骤4：测试验证**
1. 小金额测试（如0.01元）
2. 验证支付流程
3. 验证回调通知
4. 验证退款功能

## 常见误区

### ❌ 错误用法
- 在生产环境使用 `alipaysandbox` 插件
- 在沙箱环境使用生产APPID
- 混用沙箱和生产的密钥配置

### ✅ 正确用法
- 开发测试：`alipaysandbox` + 沙箱APPID + 沙箱密钥
- 生产上线：`alipay` + 生产APPID + 生产密钥
- 两个环境完全隔离，分别配置

---

## 技术原理

### 为什么要单独创建沙箱插件？

1. **配置隔离**：沙箱和生产使用不同配置，避免混淆
2. **便于切换**：可以同时启用两个通道，方便测试
3. **安全性**：沙箱密钥泄露不影响生产环境
4. **清晰标识**：插件名称明确标识环境，避免误用

### 代码复用机制

```php
// alipaysandbox 通过静态方法调用复用 alipay 的代码
class alipaysandbox_plugin {
    static public function submit(){
        return alipay_plugin::submit();
    }
}

// config.php 通过不同的 gateway_url 区分环境
// alipay: https://openapi.alipay.com/gateway.do
// alipaysandbox: https://openapi-sandbox.dl.alipaydev.com/gateway.do
```

这种设计：
- ✅ 不重复代码，便于维护
- ✅ 配置独立，不会混淆
- ✅ 可以同时使用两个环境
