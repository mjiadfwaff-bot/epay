## ⚠️ 发现的安全风险

### 🔴 高危风险

#### 1. SQL注入风险
**文件**: `cashier.php`
**代码**:
```php
$trade_no=daddslashes($_GET['trade_no']);
$row=$DB->getRow("SELECT * FROM pre_order WHERE trade_no='{$trade_no}' limit 1");
```
**问题**: 虽然使用了 `daddslashes()` 过滤，但仍然是字符串拼接SQL，存在绕过风险
**建议**: 使用参数化查询（PDO预处理）

**影响**: 攻击者可能通过构造特殊字符绕过过滤，执行恶意SQL

---

#### 2. XSS跨站脚本风险
**文件**: `pay.php`、`cashier.php`
**代码**:
```php
$sitename=isset($_GET['sitename'])?base64_decode($_GET['sitename']):'';
// 直接输出到HTML
```
**问题**: `base64_decode` 后的内容直接输出到页面，未进行HTML转义
**建议**: 使用 `htmlspecialchars()` 转义输出

**影响**: 攻击者可能注入JavaScript代码，窃取用户Cookie或会话

---

#### 3. 前端使用 `eval()`
**文件**: `paypage/js/pay.js`
**代码**:
```javascript
var jsonPayStr = eval("("+payStr+")");
```
**问题**: `eval()` 执行字符串代码，存在代码注入风险
**建议**: 使用 `JSON.parse()` 替代

**影响**: 如果payStr被污染，可能执行任意JS代码

#### 6. 会话安全
**文件**: `includes/common.php`
**代码**:
```php

```
**风险**:
- 未设置 `httponly` 和 `secure` Cookie
- 未设置会话超时

**建议**:

