# SQL注入POC测试工具使用说明

## 📋 工具概述

这是一个自动化的SQL注入漏洞测试工具，专门用于测试 `cashier.php` 文件中的 `trade_no` 参数是否存在SQL注入漏洞。

**文件位置**：`test_sql_injection.py`

---

## 🚀 快速使用

### 基本用法

```bash
# 测试目标域名
python3 test_sql_injection.py http://your-domain.com
```

### 示例

```bash
# 测试HTTP站点
python3 test_sql_injection.py http://pay.example.com

# 测试HTTPS站点
python3 test_sql_injection.py https://secure.example.com
```

## 🔍 测试项目

工具会自动执行以下4种SQL注入测试：

### 1. 基础SQL注入测试
- **测试payload**：`test' OR '1'='1`
- **检测目标**：是否返回SQL错误信息或异常响应
- **漏洞特征**：SQL语法错误、响应长度差异

### 2. GBK宽字节注入测试
- **测试payload**：`test%df' OR 1=1--`
- **原理**：利用 `%df\'` 在GBK编码下被解释为 `運'`，绕过 `addslashes()` 转义
- **漏洞特征**：SQL错误、异常数据返回

### 3. 时间盲注测试
- **测试payload**：`test' AND SLEEP(3)--`
- **原理**：如果SQL被执行，`SLEEP(3)` 会导致响应延迟3秒
- **漏洞特征**：响应时间比正常请求延迟2.5秒以上

### 4. 联合查询注入测试
- **测试payload**：`-1' UNION SELECT 1,2,3,...,20--`
- **检测目标**：是否可以执行UNION查询获取额外数据
- **漏洞特征**：页面回显数字或SQL错误

## 🛡️ 修复验证

### 如何验证修复是否成功？

1. **修复前测试**：
```bash
python3 test_sql_injection.py http://your-domain.com
# 预期：显示 "存在SQL注入漏洞"
```

2. **应用修复方案**（参考 `docs/SQL注入测试与修复.md`）

3. **修复后测试**：
```bash
python3 test_sql_injection.py http://your-domain.com
# 预期：显示 "未发现SQL注入漏洞" 或 "已修复"
```

### 修复成功的特征

工具检测到以下特征时，会判定为已修复：
- 返回 "订单号格式错误"
- 返回 "订单不存在"（且无SQL错误）
- 所有注入payload都被正确过滤

## ⚠️ 安全提示

### 合法使用声明

**重要：本工具仅用于安全测试和漏洞验证，请遵守以下原则：**

1. ✅ **授权测试**：仅对自己拥有或已获得明确授权的系统进行测试
2. ✅ **漏洞修复**：用于验证安全修复是否生效
3. ✅ **安全审计**：作为安全审计的一部分
4. ❌ **禁止攻击**：不得对未授权的系统进行测试
5. ❌ **禁止破坏**：不得利用漏洞进行数据窃取或破坏

### 测试建议

- 在**测试环境**中先进行验证
- 避免在**生产高峰期**进行测试
- 时间盲注测试会占用服务器资源，请谨慎使用
- 保留测试日志，便于追溯

## 🐛 常见问题

### Q1: 提示 "ModuleNotFoundError: No module named 'requests'"
**解决**：
```bash
pip install requests
# 或
pip3 install requests
```

### Q2: 所有测试都显示 "测试出错"
**可能原因**：
- 域名不可访问
- 网络连接问题
- 目标站点设置了防火墙/WAF

**解决**：
1. 检查域名是否正确
2. 尝试在浏览器中访问该域名
3. 检查网络连接

### Q3: 时间盲注测试总是超时
**可能原因**：
- 服务器响应慢
- 网络延迟高

**解决**：
修改超时时间（编辑脚本第112行）：
```python
response_time = requests.get(self.target_url, params=params_time, timeout=15)  # 改为15秒
```

### Q4: 修复后仍然显示存在漏洞
**可能原因**：
- 修复不完整
- 缓存问题

**解决**：
1. 检查 cashier.php 是否使用了PDO预处理
2. 检查是否有输入格式验证
3. 清除服务器缓存后重试
