# 艾玛支付插件故障排查指南

## 问题1：日志文件没有生成

### 可能原因
1. **目录权限不足** - Web服务器用户无法写入日志目录
2. **配置未正确加载** - config.php 文件路径错误
3. **log_path 配置为空** - 检查 config.php 中的 log_path 配置

### 解决方案

#### 1. 检查目录权限
```bash
# 查看当前权限
ls -la plugins/aima/inc/log/

# 修改权限（根据实际情况调整）
chmod 755 plugins/aima/inc/log/
chown www-data:www-data plugins/aima/inc/log/  # Linux/Apache
# 或
chown nginx:nginx plugins/aima/inc/log/        # Linux/Nginx
```

#### 2. 手动创建测试日志
```bash
# 进入日志目录
cd plugins/aima/inc/log/

# 创建测试文件
echo "test" > test.log

# 如果无法创建，说明权限有问题
```

#### 3. 检查 PHP 错误日志
查看 PHP 错误日志，看是否有文件写入失败的错误信息。

## 问题2：支付下单失败（未返回 pay_bankcode）

### 可能原因
1. **配置未填写** - 未在后台配置支付产品编码
2. **配置读取失败** - config.php 加载失败
3. **签名错误** - 签名算法不正确

### 解决方案

#### 1. 检查配置
登录商户后台，确认已填写：
- 网关地址（appurl）
- 商户号（appid）
- 商户秘钥（appkey）
- 支付宝产品编码（appmchid）
- 微信产品编码（appswitch）

#### 2. 查看日志文件
日志文件位置：`plugins/aima/inc/log/[日期].log`

查找关键信息：
```
====== 开始创建订单 ======
配置信息: {"pay_memberid":"M200025","pay_bankcode":"ALIPAY",...}
签名参数: {...}
请求: http://xxx/Pay_Index.html
响应: {...}
```

#### 3. 验证签名算法
使用测试脚本验证签名：
```bash
php plugins/aima/test_sign.php
```

## 问题3：回调验证失败

### 可能原因
1. **IP白名单限制** - 服务器未放行回调IP
2. **签名验证失败** - 回调参数被篡改或签名算法不一致
3. **订单号或金额不匹配** - 回调数据与订单信息不符

### 解决方案

#### 1. 检查服务器防火墙
确保已放行回调IP：`34.92.216.115`

```bash
# 查看防火墙规则
iptables -L -n | grep 34.92.216.115

# 添加白名单（示例）
iptables -A INPUT -s 34.92.216.115 -j ACCEPT
```

#### 2. 查看回调日志
```
====== 收到支付回调 ======
回调数据: {"memberid":"M200025","orderid":"...","amount":"100.00",...}
签名验证通过
支付成功：订单号=xxx, 金额=100.00, 流水号=xxx
```

#### 3. 测试回调接口
使用 Postman 或 curl 模拟回调请求：
```bash
curl -X POST http://你的域名/pay/notify/订单号/ \
  -d "memberid=M200025" \
  -d "orderid=订单号" \
  -d "amount=100.00" \
  -d "transaction_id=test123" \
  -d "datetime=2024-01-01 12:00:00" \
  -d "returncode=00" \
  -d "sign=签名值"
```

## 问题4：支付页面无法跳转

### 可能原因
1. **返回链接格式错误** - API返回的支付链接格式不正确
2. **二维码生成失败** - 前端页面错误
3. **网络问题** - 无法访问支付链接

### 解决方案

#### 1. 检查返回数据
查看日志中的 API 响应：
```json
{
  "status": 1,
  "msg": "SUCCESS",
  "h5_url": "https://...",
  "pay_url": "https://..."
}
```

#### 2. 测试支付链接
复制日志中的支付链接，在浏览器中直接访问，看是否能正常打开。

## 调试技巧

### 1. 启用详细日志
确保 `config.php` 中 `log_path` 配置正确：
```php
'log_path' => dirname(__FILE__).'/log/',
```

### 2. 查看实时日志
```bash
# Linux/Mac
tail -f plugins/aima/inc/log/$(date +%Y-%m-%d).log

# 或使用 grep 过滤
tail -f plugins/aima/inc/log/$(date +%Y-%m-%d).log | grep "ERROR\|失败"
```

### 3. 使用浏览器开发者工具
- 打开浏览器 F12 开发者工具
- 查看 Network 选项卡，观察 API 请求和响应
- 查看 Console 选项卡，查找 JavaScript 错误

### 4. 测试环境验证
先在测试环境验证功能，确认无误后再部署到生产环境。

## 常见错误码

| 错误信息 | 可能原因 | 解决方案 |
|---------|---------|---------|
| "请先配置支付宝产品编码" | 未填写支付产品编码 | 在后台配置 appmchid |
| "接口返回数据解析失败" | API返回非JSON格式 | 检查网关地址是否正确 |
| "签名验证失败" | 签名算法错误或秘钥错误 | 检查商户秘钥是否正确 |
| "订单号或金额不匹配" | 回调数据与订单信息不符 | 检查订单信息 |
| "IP验证失败" | 回调IP不在白名单 | 添加IP到白名单或清空白名单配置 |
